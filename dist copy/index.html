DATA NEVER LEAVES WITHOUT CONSENT

Building on
the context layer.

How a developer reaches their data. There are two paths, split by one question: is it your data, or someone else's? Your own ring is wide open, reachable by a realtime API or by MCP. Everyone else's runs through a closed loop.

00 · The cast

Four users, two of them build.

Everyone is really one of two personas in a private or exposed mode. This manual is for the two on the right: the people who write code.

TYPE 1

Wearer · local

Stays in the app. On-device only, nothing exported. The default state for every user.

own data · no export
TYPE 2

Wearer · connected

Same person, after granting an MCP scope to their own agent (Claude, ChatGPT, Gemini).

own data · granted
TYPE 3

Builder · self

Builds on their own ring: a tracker, a dashboard, an agent. Reaches data by realtime API or MCP. No sandbox.

own data · open access
TYPE 4

Builder · market

Ships apps other users install. The only persona that touches other people's data, and the only one in the loop.

others' data · sandboxed
01 · The deciding question

Whose data is it?

That one question decides everything. The closed-loop sandbox is not a universal law. It is the price of touching someone else's data. On your own ring there is no sandbox and no required toolchain. You get scoped access and build whatever you want, on whatever platform you want.

OWN DATA → OPEN

It's yours. Do anything.

Authenticate, grant yourself the scopes, and reach your ring through a realtime API and SDK, or through MCP for agents. Build a native iOS or Android app, a CLI, a dashboard, or your own assistant. Once the data is in your code it is yours to keep, store, or send wherever. There is no one to protect you from.

no sandbox · no bundle · no review
OTHERS' DATA → CLOSED LOOP

Not yours. Stay in the loop.

To run on other people's rings you ship a signed bundle into a sandbox on each user's phone. No open network: input arrives only through granted read tools, output leaves only through granted action tools. You never receive their raw data. This path exists only because the data is not yours.

sandbox · manifest · review

Why the rules differIt's your data, you consentedIt's not your data
Your own ring is wide open. Pull it, stream it, or hand it to an agent. The loop only closes the moment you reach for someone else's data.

One rule generates everything: data never leaves without consent. When the data is yours, you are the consent, so you get a fast, raw, realtime API with nothing fenced off. When it belongs to a stranger, the sandbox is how their consent stays enforceable instead of a promise.
